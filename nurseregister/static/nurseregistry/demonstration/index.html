Step 1: Create an easy-to-edit registry with history feature

Editing of data is provided by Django-admin. An extension ensures that all edited data is provided with a history that provides tracing possibilities and protects against data loss.

Step 2 A: Provide UXP services with UXP connector

UXP connector is set up to connect directly to the registry's database. Multiple services can be created by running different SQL queries against the database.

Step 2 B: Provide access through UXP REST interface integration

Step 2 B.1

Provide a REST interface in the registry application.

Step 2 B.2

Expose the REST interface (or parts of it) as UXP services using UXP security server. Multiple services can be created by exposing multiple REST URLs through the security server.

Step 3: Consume the UXP services

Consume the exposed services in the target application. Multiple services can be consumed using the same security configuration.
